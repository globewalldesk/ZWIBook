About ZWIBook

Built by the Knowledge
Standards Foundation

This app, which we call "ZWIBook," is a versatile and user-friendly digital library application that allows users to access a vast collection of texts from Project Gutenberg. With over 69,000 texts, primarily books published before 1920, ZWIBook showcases the functionality of the ZWI file format. The application aims to demonstrate the capabilities of the ZWI format and to provide donors of the Knowledge Standards Foundation a compelling reason to support our efforts. Distributed on a 128MB thumb drive, ZWIBook is a bespoke software solution created by Larry Sanger with the help of AI tools, with assistance from Henry Sanger.

↑ The Reader: read a book

  • Click any search result title to go to the book.
  • Change font size and style under the "Text and View" menu. These choices will be saved.
  • Your last-read location is automatically saved.
  • Images are clickable and saveable.
  • To add bookmarks, hover over a paragraph. Click on the outlined bookmark in the upper left.
  • To return to a bookmark, click on the bookmarks icon [image] in the header.
  • All books opened are added to the "Books Viewed" tab in the Bookshelf (see below).
  • If you specially want to save a particular book (because you're reading it, for example), click the "save book" button [image].

Special and unexpected features:

  • Long books, like the Bible, and those with many pictures, like Pride and Prejudice, can take a few seconds to load, due to processing. (This is not done in advance, in order to preserve your copy of each file without any changes.)
  • Merge lines/Separate lines switch [image]. Plaintext books (that is, where Project Gutenberg doesn't have HTML) were rendered into nicely-formatted text as well as we could manage, by combining lines. Most of these should look rather good. For some, however, you might want to use the original line breaks. Poetry and song should preserve line breaks by default.

↑ Search: find books

  • Search title, author, or both; does not search text, but you can search on the page (see below).
  • Results are by relevance, but if you press the [sorting: image] button, you can re-sort the results by Author, Title, and Date.
  • All results are shown on the same page.

Special and unexpected features:

  • If there are too many results or search terms, you will be asked to refine the search.
  • Old-fashioned uppercase "OR" searches are supported. For example, the title search "Hume OR Kant" results in all books by Hume plus all books by Kant. ("AND" is the default, i.e., usually, results must include all search terms.)

↑ Book Categories

  • Organized by Library of Congress codes, making it easy to browse through a vast array of subjects systematically.
  • Each primary category links to subcategories, helping you narrow down to specific topics of interest.
  • Each category and subcategory page displays the number of books available, giving a clear indication of the library size in each section.
  • Books within each category and subcategory can be sorted by author, title, or date. A default sort option is available to revert to the initial order.
  • Users can easily search within category and subcategory pages using 'Find on Page', helping to quickly locate specific books or authors within large categories.

↑ Bookshelf: manage your selections

  • Opening any book automatically appends it to the top of the "viewed" list, ensuring easy access to recently read materials.
  • Users can switch between tabs to see books they have viewed and books they have saved for future reading.
  • Each tab has a "Remove All" button that allows users to clear all books from the respective list with a single click. The button is displayed when the number of books in the list reaches 20 or more.
  • Books can be sorted by author, title, or date. The sorting preference is saved and applied automatically on page load. The sort dropdown can be accessed from the header, and the sorting type is highlighted in the dropdown menu.
  • Users can delete individual books from the "viewed" or "saved" list by clicking the trash icon next to each book entry.

↑ To Verify a ZWI File

To ensure the integrity and authenticity of a ZWI file, follow these steps:

  • Grab the `token` from the signature.json file within the ZWI.
  • Visit https://did.dara.global/gutenberg/#publish and copy the `publicKeys`, which is:
{
    "kty":"EC",
    "crv":"P-384",
    "x":"1beexcFBjN6FXYN6Q5mqeOI3N4D2BH3SbP3_81kALR9kW2ZbE5QJ4yCcjE0OcXWy",
    "y":"-jMdfTnpnCCNBrtgFwrluaH5Pz3v0YYNzucC0aezTdxaUrLriisYUAHZrx_b5CD4",
    "alg":"ES384",
    "kid":"did:psqr:did.dara.global/gutenberg#publish"
}
  • On JWT.io, drop the `token` into the "Encoded" field on the left.
  • Drop the `publicKeys` into the "verify signature" area on the lower right.
  • This process establishes that, according to the DID:PSQR and JSON Web Token protocols, only the private key that matches dara.global's public key (at the above address) could have generated that particular token.
  • Some further steps can, in a similarly rigorous way, establish that only the same private key could take all the content of the ZWI file as input and generate that token as output.

↑ Notes and Disclaimers
